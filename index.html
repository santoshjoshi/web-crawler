web-crawler

Crawling the Web using Apache camel

Download this project as a .zip file Download this project as a tar.gz file

About the application:

The application is all about harnessing the capabilities of below components to create a web crawler that can crawl any site. Here we are mainly interested in getting the pin code of various areas/cities across India from a site called javataskforce.com.

This Application uses:

  1. Apache CXF:

    For Making rest/http request to remote site and fetch the page.

  2. JPA/Hibernate with HSQLDB:

    Using JPA with hibernate to populate the HSQLDB with the information we are interested in

  3. Apache Camel:

    Coordinating between Apache CXF/JPA, leveraging many task that requires a lot of coding, creating routes that states what to do and when to do

Overview:

The Whole application is interconnected by camel routes, which are as follows (all routes are defined in camel-routes.xml)

  • states-invoke-route 1. invoke the crawler sevice to get state

    1. parse/process the states obtained
    2. put them in another Queue
    3. store the states into HSQLDB

  • districts-invoke-route

    1. pool the DB( HsqlDb) and get the State with some particular status(INITIAL)
    2. update the status of State to (IN_PROCESS)
    3. invoke the crawler service to obtain districts for state
    4. parse/process the districts obtained
    5. put them in another Queue
    6. store the districts into HSQLDB
    7. mark the state as processed (COMPLETED)

  • pincode-invoke-route

    1. pool the DB( HsqlDb) and get the District with some status(INITIAL)
    2. update the status of District to (IN_PROCESS)
    3. invoke the crawler service to obtain areas/cities for the fetched District
    4. parse/process the areas/cities obtained
    5. put them in another Queue
    6. store the area/city into HSQLDB
    7. mark the district as processed (COMPLETED)

In a nutshell there are some routes which invokes a crawl url , the result of which is parsed/processed and then saved into the crawl Database

Configuring the application:

Scheduling configuration:

  1. Open camel-routes.xml and change the cron in statement to a suitable time quartz://myGroup/myTimerName?cron=30 06 00 03 08 ? 2016
  2. Open the camel-routing-policy.xml and schedule time for
    1. districts-invoke-route
    2. pincode-route-policy

Running the application:

  1. mvn clean install
  2. run the HSQL Database
  3. Go to hsqldb folder Windows: click db_sart.bat Unix/Linux: sh db_start.sh
  4. mvn exec:exec
    This will execute com.javatask.Run

Monitoring :

  1. Using Jconsole Application can be monitored using Jconsole at port 8764 ( port can be changed by changing the jmx setting in pom.xml)

    <argument>-Dcom.sun.management.jmxremote</argument>
    <argument>-Dcom.sun.management.jmxremote.authenticate=false</argument>
    <argument>-Dcom.sun.management.jmxremote.ssl=false</argument>
    <argument>-Dcom.sun.management.jmxremote.port=8764</argument>

    open Jsonsole and add service:jmx:rmi:///jndi/rmi://127.0.0.1:8764/jmxrmi

  2. Monitoring HSQL DB

    • Windows Client

      execute db_client.bat available in hsqldb folder
